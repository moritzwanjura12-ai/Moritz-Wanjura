About this project

I built this toolkit because I wanted to understand the core models of quantitative finance not just theoretically, but interactively. Every model here is something I worked through properly — the maths, the implementation, the edge cases. Being able to drag a slider and instantly see how Delta changes, or watch 200 GBM paths animate in real time, makes these concepts click in a way that reading a textbook never quite does.

Everything runs in a single HTML file — no backend, no login, no dependencies beyond Chart.js and Three.js. Each tool has a built-in explanation (click ⓘ Explain model) covering the theory, the formulas and how to use it.

Black–Scholes Model

Analytical options pricing formula · Fischer Black & Myron Scholes, 1973

What is it?

The Black–Scholes model is the foundational formula for pricing European options. It assumes the stock price follows a geometric Brownian motion — continuously and randomly fluctuating with a fixed drift and fixed volatility. From this assumption, an analytically exact price for calls and puts can be derived without simulation.

Core formulas

C = S·e−qT·N(d₁) − K·e−rT·N(d₂)
P = K·e−rT·N(−d₂) − S·e−qT·N(−d₁)

d₁ = [ln(S/K) + (r − q + ½σ²)·T] / (σ·√T)
d₂ = d₁ − σ·√T

S = Spot, K = Strike, T = Maturity, r = risk-free rate, σ = volatility, q = dividend yield, N() = normal CDF

Example

Scenario
S=52, K=50, T=0.25
Stock above strike, 3 months to expiry
Parameters
r=12%, σ=30%
High rate + moderate volatility
Call result
≈ 5.06
Option is in-the-money (ITM) — high intrinsic value
Put result
≈ 1.54
Put-call parity confirms both prices

Interpreting the Greeks

Delta (Δ) shows how much the option price changes per €1 move in the underlying. Gamma (Γ) measures curvature — how fast Delta itself changes. Vega (ν) is critical: a 1% rise in volatility increases the option price by Vega. Theta (Θ) shows daily time decay — options lose value every day even if the stock doesn't move.

How to use

  1. Enter Spot price (S) — current market price of the stock
  2. Enter Strike price (K) — the option's exercise price
  3. Set Maturity (T) in years — e.g. 3 months = 0.25
  4. Set Risk-Free Rate (r) % — e.g. current central bank rate
  5. Set Volatility (σ) % — historical or implied vol
  6. Click Calculate → Call/Put price appears instantly
  7. Explore tabs: Greeks, 2D Chart (7 modes), and 3D Surface

Model limitations

Black–Scholes assumes constant volatility — in reality a volatility smile exists. It only applies to European options (no early exercise). For American options or volatility surfaces, Binomial Tree or Heston models are more accurate.

Monte Carlo Simulation

Stochastic options pricing via path simulation

What is it?

Monte Carlo prices options by simulating thousands of possible price paths. Each path follows geometric Brownian motion. At expiry, the payoff is calculated for each path — the discounted average of all payoffs gives the option price. The method is universally applicable, including for path-dependent options (Asian, Barrier, etc.).

Core formula

S(t+Δt) = S(t) · exp[(r − ½σ²)Δt + σ·√Δt · Z]

C ≈ e−rT · (1/N) · Σ max(S_T − K, 0)

Z ~ N(0,1) · standard normal · convergence error ∝ 1/√N

Example

50 paths, 252 steps
Rough estimate
Fast, but high standard error (~15%)
200 paths, 252 steps
Good estimate
Converges close to the BS price

Interpretation

The standard error shows the uncertainty of the MC price. More paths → smaller error, but compute time grows with √N. The BS benchmark serves as reference: for European options, MC should converge to BS.

How to use

  1. Enter market parameters (S, K, T, r, σ) as in Black–Scholes
  2. Select option type: Call or Put
  3. Paths slider: more paths = more accurate but slower
  4. Steps slider: 252 steps corresponds to one trading year
  5. Click Run Simulation → paths are animated live on the canvas
  6. Red line = last simulated path · Dashed line = Strike K

Binomial Tree Model

Discrete options pricing · Cox, Ross & Rubinstein, 1979

What is it?

The Binomial Tree model divides the life of an option into discrete time steps. At each step, the price can either move up by factor u or down by factor d. Starting from today's price, a tree of all possible paths is built. Option values are computed backwards — from terminal payoffs back to the root. Crucially, for American options, each node compares whether immediate exercise is more profitable than holding.

Core formulas

u = eσ√Δt d = 1/u Δt = T/N
p = (erΔt − d) / (u − d) ← risk-neutral probability

V_node = e−rΔt · [p · V_up + (1−p) · V_down]
American: V_node = max(V_hold, Intrinsic Value)

Node colour guide

Green nodes
Early exercise
For American puts: immediate exercise beats holding
Red nodes
In the money
Positive payoffs at the terminal nodes
N = 1–5
Rough approximation
Useful for visualising the logic of the model
N = 10
Good approximation
Converges noticeably towards the BS price

How to use

  1. Enter parameters: S, K, T, r, σ as usual
  2. Choose Steps N: 1–10 (small = clearer tree, large = more accurate)
  3. Select Call or Put, then European or American
  4. Click Build Tree → tree is drawn with colour-coded nodes
  5. Top number = stock price, bottom number = option value at each node
  6. Parameter table below shows u, d, p and discount factor

Value at Risk & CVaR

Portfolio loss measures · Regulatory standard (Basel III)

What is it?

VaR answers: "What is the maximum I can lose over the next h days with probability α%?" CVaR (Expected Shortfall) goes further: "What is the average loss when VaR is exceeded?" CVaR is more coherent and increasingly preferred by regulators.

Core formulas

Parametric: VaR = −(μ_p − z_α · σ_p) · √h · W
CVaR = (μ_p · W) + σ_p · W · φ(z_α)/α · √h

σ_p² = Σ Σ w_i · w_j · σ_i · σ_j · ρ_ij

z_α = normal quantile (99% → z = 2.326) · φ() = normal PDF · W = portfolio value

Three methods compared

Parametric
Fast & analytical
Assumes normality — underestimates tail risks (fat tails)
Monte Carlo
Flexible & precise
Simulates 50,000+ scenarios, no distribution assumption needed
Historical
Realistic tails
Based on actual past market movements
Loss histogram
Red zone = tail
Losses beyond VaR are highlighted in red

Interpretation

A 99% VaR of €5,000 means: with 99% probability, the portfolio will not lose more than €5,000 in one day. The CVaR might then be €8,000 — the average loss in the worst 1% of scenarios.

How to use

  1. Enter assets: name, weight %, expected return %, and volatility %
  2. Weights don't need to sum to 100 — they are normalised internally
  3. Confidence level: 95% (internal), 99% (standard), 99.9% (stress test)
  4. Horizon: 1 day (trading), 10 days (regulatory), 252 days (annual VaR)
  5. Select method → click Calculate VaR & CVaR
  6. Comparison table shows all three methods side by side

Efficient Frontier & Portfolio Optimisation

Mean-variance optimisation · Harry Markowitz, 1952

What is it?

The Efficient Frontier is the set of all portfolios that achieve the highest return for a given level of risk — or the lowest risk for a given return. No portfolio above the frontier is achievable, and no portfolio below it is optimal. Through diversification (correlation < 1), portfolio risk falls below the weighted average of individual risks.

Core formulas

E[R_p] = Σ w_i · μ_i
σ_p² = Σ Σ w_i · w_j · σ_i · σ_j · ρ_ij

Sharpe Ratio = (E[R_p] − r_f) / σ_p

w_i = weights (sum to 1) · ρ_ij = correlation between asset i and j

Key points in the chart

★ Max Sharpe (yellow)
Tangency portfolio
Best risk-adjusted return — intersection of Capital Market Line and frontier
◆ Min Variance (green)
Global minimum variance
Lowest achievable risk, independent of return
Capital Market Line
Red dashed line
Combination of risk-free asset and tangency portfolio
Point colour scale
Blue = high Sharpe
Each point is a randomly sampled portfolio from 5,000 simulations

How to use

  1. Enter assets: name, expected annual return %, annual volatility %
  2. Edit the correlation matrix — values between −1 and 1
  3. Set risk-free rate r_f (e.g. current central bank rate)
  4. Portfolios to sample: more = more accurate frontier (but slower)
  5. Short selling: "No" = long only (more realistic for retail investors)
  6. Click Optimize Portfolio → scatter plot, frontier line and weights donut appear

Bond Pricer

Bond pricing · Duration · Convexity · Sensitivity analysis

What is it?

A bond pays periodic coupons and returns face value at maturity. The fair price is the present value of all future cash flows, discounted at the market yield (YTM). When market yields rise, bond prices fall — and vice versa. This inverse relationship is fundamental for all fixed income investors.

Core formulas

P = Σ [C / (1 + y/m)^t] + F / (1 + y/m)^n

D_Mac = Σ [t · PV(CF_t)] / P ← Macaulay Duration (years)
D_Mod = D_Mac / (1 + y/m) ← price sensitivity
DV01 = D_Mod · P · 0.0001 ← € change per basis point

C = coupon · F/m, y = YTM, m = frequency, n = periods

Example

Coupon 5%, YTM 6%
Price < 100
Bond trades below par (discount bond)
Coupon 5%, YTM 4%
Price > 100
Bond trades above par (premium bond)
Duration 7 years
High sensitivity
+1% rate rise → price loss ≈ 7%
DV01 = €65
Per basis point
1bp yield change = €65 price change

How to use

  1. Enter face value (F), coupon rate %, maturity in years and YTM %
  2. Frequency: Semi-Annual (US standard), Annual (EU government bonds)
  3. Optionally enter market price P* → YTM is back-calculated
  4. Click Price Bond → price, duration, convexity appear
  5. Click through tabs: Cash Flows → Price vs YTM → Duration Chart → Sensitivity

Yield Curve Builder

Term structure of interest rates · Spreads · Recession indicator

What is it?

The yield curve shows the yields of bonds with the same credit quality across different maturities. Normally it slopes upward — longer maturities yield more (liquidity premium). When the curve inverts (short maturities yield more than long ones), it has historically been one of the most reliable recession signals.

How to use

  1. Enter yields for all 10 maturities (in % p.a.)

  3. Click Plot Yield Curve → curve is drawn
  4. Shape (Normal/Flat/Inverted) and spreads are calculated automatically

Geometric Brownian Motion — dS = μS dt + σS dWⓘ Explain model

Geometric Brownian Motion

The standard model for stock prices · Foundation of Black–Scholes

What is it?

GBM is the most important stochastic model in finance. It describes stock prices as continuously but randomly fluctuating — with a drift μ (growth trend) and diffusion σ (random noise). The key advantage: GBM always guarantees positive prices (S > 0) because the price is updated multiplicatively.

Core formula

dS = μ·S·dt + σ·S·dW ← stochastic differential equation

Exact solution:
S(t) = S₀ · exp[(μ − ½σ²)·t + σ·W(t)]

E[S(T)] = S₀ · eμT ← theoretical expected value
W(t) = Brownian motion (Wiener process)

Why μ − ½σ²?

The ½σ² term is the Itô correction from stochastic calculus. Intuitively: since the logarithm is concave, the average of log-returns always lies slightly below μ. The terminal histogram shows a log-normal distribution — the typical distribution of stock prices.

How to use

  1. S₀ = starting stock price (e.g. 100)
  2. μ = annual drift in % (e.g. 8% for average equity market)
  3. σ = annual volatility in % (DAX ≈ 20%, tech stocks ≈ 40%)
  4. T = time horizon in years, N = time steps (252 = one trading year)
  5. Paths = number of simulated paths (red line = last path)
  6. Simulate → paths appear, terminal histogram below shows log-normal shape

Standard Brownian Motion — dW ~ N(0, dt)ⓘ Explain model

Standard Brownian Motion

Wiener process · The fundamental stochastic process

What is it?

Brownian motion (Wiener process) is the mathematical building block of all stochastic processes in quantitative finance. It starts at 0, moves continuously, and has independent normally distributed increments. All other processes — GBM, OU, Jump Diffusion — are built on top of it.

Core formula

dW ~ N(0, dt) ← infinitesimal increments

Properties:
W(0) = 0
E[W(t)] = 0 ← no drift, no trend
Var[W(t)] = t ← variance grows linearly with time
W(t) ~ N(0, √t) ← normally distributed at every point in time

Interpreting the statistics

Mean W(T) ≈ 0
No trend
BM has no systematic drift — it is a fair process
Var[W(T)] ≈ T
Variance = time
Fundamental property: the longer the process runs, the more it disperses

How to use

  1. T = time horizon in years (e.g. 1)
  2. N = time steps (more = smoother paths)
  3. Paths = number of simulated paths
  4. Simulate → paths appear, statistics verify E[W]=0 and Var≈T

Ornstein–Uhlenbeck — dX = θ(μ − X)dt + σ dWⓘ Explain model

Ornstein–Uhlenbeck Process

Mean-reverting process · Interest rates, spreads, commodities

What is it?

The OU process describes quantities that fluctuate around a long-run mean μ and are continuously "pulled" back towards it. Unlike GBM, the process can go negative and has no unbounded growth. It is ideal for interest rates (Vasicek model), credit spreads, volatility and commodities that show mean-reverting behaviour.

Core formula

dX = θ·(μ − X)·dt + σ·dW

θ·(μ − X) = mean-reversion force
When X > μ: force pulls downward
When X < μ: force pulls upward

Half-life = ln(2) / θ
E[X(T)] = μ + (X₀ − μ)·e−θT

Parameter intuition

θ small (0.1–0.5)
Slow reversion
Process wanders far from mean, takes long to return
θ large (5–10)
Fast reversion
Paths cling to the mean — little deviation possible
Half-life
ln(2) / θ
Time for half the deviation from the mean to be absorbed
Long-run variance
σ² / (2θ)
Equilibrium variance — the process converges to this

How to use

  1. X₀ = starting value (set away from μ to observe reversion)
  2. μ = long-run mean (e.g. 0 for a spread)
  3. θ = mean-reversion speed (e.g. 2 = moderate reversion)
  4. σ = volatility of the noise term
  5. Simulate → half-life in the stats strip shows how quickly the process returns to the mean

Merton Jump Diffusion — dS = μS dt + σS dW + S dJⓘ Explain model

Merton Jump Diffusion

GBM + Poisson jumps · Robert Merton, 1976

What is it?

The Merton model extends GBM with sudden jumps — e.g. earnings surprises, profit warnings or market crashes. The price mostly behaves like GBM, but occasionally there are abrupt up or down moves. These jumps follow a Poisson process (random arrival times) and have random sizes (log-normally distributed).

Core formula

dS = μ·S·dt + σ·S·dW + S·dJ

N(t) ~ Poisson(λ·t) ← number of jumps up to time t
Jk ~ log-N(μ_j, σ_j²) ← jump sizes

μ_adj = μ − λ·(eμⱼ+½σⱼ²−1) ← drift compensation

Parameter intuition

λ (intensity)
Jumps per year
λ=3: average of 3 jumps per year — roughly one every 4 months
μⱼ (jump mean)
Negative value
−5% = jumps are on average downward shocks (fat left tail)
σⱼ (jump vol)
Spread of jumps
High σⱼ = sometimes small, sometimes very large jumps
Terminal histogram
Fat tails
Compared to GBM: more extreme values on both sides

How to use

  1. Enter GBM parameters: S₀, μ, σ, T, N, Paths
  2. λ = expected jumps per year (e.g. 3–5 for a single stock)
  3. μⱼ = average jump return in % (negative = crash risk)
  4. σⱼ = volatility of jump sizes
  5. Simulate → stats show average jumps vs expected value λ·T
